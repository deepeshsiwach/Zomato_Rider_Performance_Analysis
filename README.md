# 🚴‍♂️ Zomato Rider Performance Analysis

### 📊 Project Overview
This project analyzes **Zomato delivery riders’ performance** to identify operational inefficiencies, evaluate rider productivity, and explore key performance trends such as delivery times, customer ratings, and engagement.

The goal was to derive **actionable insights** for improving delivery efficiency and rider satisfaction.

---

### 🧠 Objectives
- Analyze rider efficiency and identify factors affecting performance.  
- Evaluate the relationship between **delivery time, ratings, and earnings**.  
- Highlight trends and recommendations to optimize operations.  
- Build interactive **Power BI dashboards** for visual insights.

---

### 🗂️ Dataset Information
**Data Used:**  
- Rider Personal Details  
- City Information  
- Total Calls & Deliveries  
- Attendance & Active Time  
- Salary & Ratings  

---

### 🧰 Tools & Technologies
| Tool | Purpose |
|------|----------|
| **Python** | Data Cleaning, EDA |
| **Power BI** | Interactive Dashboard Creation |
| **Excel** | Pre-processing and Data Validation |

---

### 📈 Key Insights
- Riders with **higher active hours** generally achieve **better ratings** and earnings.  
- **Customer satisfaction** drops sharply beyond a certain delivery time threshold.  
- Certain cities show **higher no-delivery (NPT)** rates due to traffic and operational delays.  
- Identified opportunities for **10–15% performance improvement** through scheduling adjustments.
